# 변수 (22.02.10 작성)

```markdown
오늘은 변수에 대해서 다시 이해하고 공부했다.

변수는 코드를 짤 때 무조건 사용한다고 본다.

지금까지는 그냥 데이터를 저장할 때 사용한다고만 생각했지만 이 책에서는 아주 상세하게 변수에 대해 설명한다.

변수(variable) 란 무엇이고 왜 필요한가?

변수(variable)는 프로그래밍언어에서 데이터를 관리하기 위한 핵심 개념이다.

변수(variable)는 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 

붙인 이름을 말한다.

자바스크립트 엔진은 5 + 10이라는 식의 의미를 해석하면 좌변과 우변의 숫자 값, 즉 피연산자(operand) 를 기억한다. 사람은 계산과 기억을 두뇌에서 한다. 저번 글에서 설명했지만 컴퓨터는 매우 수동적이다.

CPU를 사용해 연산하고 메모리를 사용해 데이터를 기억한다.

메모리 구조

메모리는 이렇게 생겼다.

메모리 셀은 하나의 크기당 1바이트(8비트)이다.

즉 1바이트 단위로 데이터를 저장하거나 읽어 들인다.

각 셀은 고유의 메모리 주소를 가진다.

메모리 주소란 메모리 공간의 위치를 나타낸다. 0부터 시작해서 메모리의 크기만큼 정수로 표현된다.

컴퓨터는 모든 데이터를 2진수로 처리한다. 데이터의 종류 (숫자, 텍스트, 이미지, 동영상 등) 과 상관없이 모두 2진수이다. <!--매우 중요--!>

메모리의 값을 저장해도 10 + 5의 값 15를 사용할 수 없다.

재 사용하려면 메모리 주소의 값에 접근을 해야 한다.

하지만 이 접근은 매우 위험한 접근이다. (지뢰밭에 들어간다고 생각하자)

왜 위험한 접근일까?

치명적인 오류를 발생시킬 가능성이 높은 일이다.

운영체제가 사용하고 있는 값을 변경하면 시스템을 멈추게 하는 치명적인 오류가 발생할 수도 있다.

자바스크립트는 개발자의 직접적인 메모리 제어를 허용하지 않는다.

코드가 실행되기 전에는 값이 저장된 메모리 주소를 알 수 없으며 알려주지도 않는다.

따라서 프로그래밍 언어는 기억하고 싶은 값을 메모리에 저장하고, 저장된 값을 읽어 들여 재사용하기 위해 변수라는 메커니즘을 제공한다.

변수는 값의 위치를 가리키는 상징적인 이름이다.

변수의 선언 과 할당

var userName = 'Kim';
var userAge = 25;

위에 코드에서 var userName 은 변수를 선언한다는 뜻이다.

(나는 userName 이라는 변수를 선언할 거야!)

var userName에 'kim'이라는 값을 넣는 것을 할당이라고 한다.

(userName이라는 변수에는 kim을 저장하고 싶어)

kim과 25는 변수 값이라고 한다.

이렇게 생성된 값은 메모리 공간에 저장된다.

코드는 컴퓨터에게 내리는 명령이지만 개발자를 위한 문서이다.

개발자의 의도를 나타내는 명확한 네이밍은 코드를 이해하기 쉽게 만들고 협업과 품질 향상에 도움을 준다.

변수 이름은 첫아이 이름을 짓듯이 심사숙고해서 지어야 한다
```
